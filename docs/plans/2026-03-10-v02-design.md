# ML/RL OS v0.2 Design Document

> **Date:** 2026-03-10
> **Status:** Approved
> **Scope:** Full v0.2 — RL engine, hyperparameter tuning, PostgreSQL backend, streaming inference, sequence models, HTML reports

## 1. Overview

v0.2 extends the complete v0.1 supervised learning pipeline with 7 new capabilities:

| # | Module | Location | What It Does |
|---|---|---|---|
| 1 | RL Engine | `rl/` | Custom PPO/DQN training against SimOS via WebSocket |
| 2 | SimOS Environment | `rl/environment.py` | Pure Python env adapter (no Gymnasium dependency) |
| 3 | Hyperparameter Tuning | `experiment/runner.py` | Wire existing `tuning.py` Optuna integration into runner |
| 4 | PostgreSQL Backend | `storage/` | Optional DB storage (file remains default) |
| 5 | Streaming Inference | `streaming/` | WebSocket endpoint for real-time predictions |
| 6 | Sequence Models | `models/algorithms/lstm.py` | LSTM for time-series via algorithm registry |
| 7 | HTML Reports | `evaluation/html_export.py` | Standalone HTML report with embedded charts |

## 2. RL Engine (`src/mlrl_os/rl/`)

### 2.1 Design Philosophy

No Gymnasium dependency. Pure Python classes with clear contracts:
- `reset() -> Observation`
- `step(action) -> (Observation, reward, done, info)`
- Pluggable reward functions, observation builders, action handlers
- Transparent, inspectable, subclassable

### 2.3 Environment (`environment.py`)

```python
class SimOSEnvironment:
    """Pure Python RL environment backed by SimOS WebSocket stepping."""

    def __init__(
        self,
        simos_url: str,                  # ws://localhost:8000/simulations/live
        template: str,                   # "healthcare_er"
        reward_fn: RewardFunction,       # pluggable reward computation
        obs_builder: ObsBuilder,         # pluggable observation extraction
        action_handler: ActionHandler,   # maps actions -> SimOS commands
        seed: int = 42,
        api_key: str = "sk-premium-test-003",
        max_steps_per_episode: int = 10_000,
    ): ...

    def reset(self) -> Observation:
        """Start new simulation via SimOS REST, connect WebSocket, return initial obs."""

    def step(self, action: Action) -> tuple[Observation, float, bool, dict]:
        """Apply action, step SimOS, compute reward, return transition."""

    def close(self) -> None:
        """Disconnect WebSocket, cleanup."""
```

**Reset flow:**
1. POST `/api/v1/simulations` with template config (REST)
2. Connect WebSocket to `/simulations/live/{job_id}`
3. Send `{"command": "pause"}`
4. Read initial snapshot -> `obs_builder.build(snapshot)`
5. Return observation

**Step flow:**
1. `action_handler.apply(action, websocket)` (e.g., routing override)
2. Send `{"command": "step", "count": N}`
3. Receive event_batch / snapshot
4. `obs = obs_builder.build(state)`
5. `reward = reward_fn.compute(prev_state, action, state)`
6. Check termination
7. Return `(obs, reward, done, info)`

### 2.7 PPO (`algorithms/ppo.py`)

Custom implementation:
- Actor-critic MLP (shared or separate backbones)
- Clipped surrogate objective
- Generalized Advantage Estimation (GAE)
- Mini-batch updates over rollout buffer
- Entropy bonus for exploration
- Seeded via `seed_hash("ppo", global_seed)`

### 2.8 DQN (`algorithms/dqn.py`)

Custom implementation:
- MLP Q-network + target network
- Epsilon-greedy exploration (decaying)
- Experience replay (uniform or prioritized)
- Huber loss
- Soft target updates (polyak averaging)
- Seeded via `seed_hash("dqn", global_seed)`

## 5. Streaming Inference (`src/mlrl_os/streaming/`)

### 5.1 ML Model Predictions

```python
@router.websocket("/ws/v1/predict/{model_id}")
async def ws_predict(websocket: WebSocket, model_id: str):
    """Accept observation batches, return predictions."""
    model = model_registry.load(model_id)
    await websocket.accept()
    while True:
        data = await websocket.receive_json()
        # {"observations": [[f1, f2, ...], ...]}
        predictions = model.predict(np.array(data["observations"]))
        await websocket.send_json({
            "predictions": predictions.tolist(),
            "model_id": model_id,
            "timestamp": time.time(),
        })
```

### 5.2 RL Policy Actions

```python
@router.websocket("/ws/v1/policy/{policy_id}")
async def ws_policy(websocket: WebSocket, policy_id: str):
    """Accept states, return actions from trained RL policy."""
    policy = rl_registry.load(policy_id)
    await websocket.accept()
    while True:
        data = await websocket.receive_json()
        state = np.array(data["state"])
        action = policy.select_action(state, explore=False)
        await websocket.send_json({
            "action": int(action),
            "policy_id": policy_id,
        })
```

## 12. Constraints Carried from v0.1

- Seeded RNG per component (`seed_hash()`)
- No training without validation (RL configs validated too)
- Immutable artifacts (policies get new IDs, not modified)
- No heavy imports at module level (PyTorch lazy-loaded)
- SimOS is data, not code (access via REST/WebSocket only)
- Builder produces YAML (RL experiments use same YAML path)
- All errors at once (validation returns all errors)
